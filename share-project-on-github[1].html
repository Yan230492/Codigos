Compartir un proyecto en GitHub

Para compartir su proyecto, puede hacerlo disponible públicamente en GitHub®. Para ello debe tener una cuenta de GitHub.

Puede compartir cualquier proyecto. El uso compartido añade el control de versiones de Git™ al proyecto abierto. Si su proyecto ya tiene control de versiones, el uso compartido reemplaza la configuración del control de versiones por Git, y el repositorio remoto del proyecto es GitHub.

Nota

Si no desea cambiar su control de versiones actual en el proyecto abierto, comparta una copia del proyecto en su lugar. Para crear una copia para compartir, consulte Archive Projects.

  1. Con un proyecto cargado, en la pestaña Project, seleccione Share > Change Share Options.

  2. Añada la opción GitHub a su menú Share. En el cuadro de diálogo Manage Sharing, seleccione GitHub y haga clic en Close.

  3. Seleccione Share > GitHub.

  4. En el cuadro de diálogo Create GitHub Repository, introduzca su nombre de usuario y token de acceso personal de GitHub y edite el nombre del nuevo repositorio. Haga clic en Create.

    Se muestra una advertencia para que confirme que desea crear un repositorio público y modificar la ubicación del repositorio remoto del proyecto actual. Para continuar, haga clic en Yes.

  5. El cuadro de diálogo Create GitHub Repository muestra la dirección URL de su nuevo repositorio. Haga clic en el enlace para ver el nuevo repositorio en el sitio web de GitHub. El repositorio contiene la confirmación inicial de sus archivos de proyecto.

  6. El control de versiones en su proyecto actual ahora hace referencia al nuevo repositorio de GitHub como repositorio remoto. Para usar el proyecto con el nuevo repositorio, en el cuadro de diálogo Create GitHub Repository, haga clic en Reload Project.

En el proyecto, puede encontrar la dirección URL de su repositorio remoto en la pestaña Project, en Source Control, usando el botón Git Details.

Si aún no ha configurado Git, necesita algunos pasos de configuración adicionales para poder fusionar bifurcaciones. Puede usar otras funcionalidades de Git sin ninguna instalación adicional. Consulte Set Up Git Source Control.
